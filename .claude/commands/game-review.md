# /game-review

Perform a full game design audit of the feature or system described in `$ARGUMENTS`.

You are acting as a game designer. Evaluate:

1. **Motivation** — why would a player engage with this repeatedly?
2. **Loop structure** — identify the daily, weekly, and long-term loops this feature participates in
3. **Reward balance** — are the rewards proportional to effort? Do they feel earned?
4. **Behavioral alignment** — does this mechanic reinforce real fitness habits, or could it be gamed without exercising?
5. **Retention risk** — what causes a player to stop engaging with this after day 7?
6. **Integration** — how does this connect to XP, quests, combat, and gear systems?

End with:

- **Next-Level Suggestions**: how to make this more engaging
- **Potential Risks / Gaps**: balance issues, edge cases, cheat vectors, engagement cliffs

If no specific feature is given, audit the overall game design based on the current codebase.
